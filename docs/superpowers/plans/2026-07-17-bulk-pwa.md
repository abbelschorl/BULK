# Bulk PWA Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the Bulk bulking tracker as an installable PWA (`web/`) with full parity to the Swift app minus HealthKit and barcode scanning, deployed to GitHub Pages.

**Architecture:** Vite + React + TypeScript SPA. Pure logic functions ported 1:1 from `Bulk/Logic/*.swift` (unit-tested with Vitest), IndexedDB persistence via `idb`, localStorage settings, five-tab mobile-first UI, `vite-plugin-pwa` for installability/offline.

**Tech Stack:** Vite, React 18, TypeScript, idb, vite-plugin-pwa, Vitest.

## Global Constraints

- Spec: `docs/superpowers/specs/2026-07-17-bulk-pwa-design.md`. The Swift sources in `Bulk/` are the authoritative behavior spec for every port; match semantics exactly unless the spec lists a deviation.
- Accepted deviations: JS numbers instead of `Decimal` (round at display: integer kcal, 1 decimal macros); no HealthKit; no camera barcode scanning.
- Goals are minimums only — below reads red/orange, at-or-above reads green. Never add an upper warning range.
- Log entries and saved-meal components snapshot per-100g nutrition at creation; history is immutable.
- Day keys: local-timezone start of day serialized `YYYY-MM-DD`.
- Backup JSON must round-trip with the Swift `ExportImportService.Backup` v1 schema (same field names; dates ISO 8601).
- Dark-first design; defaults: 3000 kcal, 150 g protein, 3000 ml water, 0.25 kg/week.
- All work happens in `web/` plus `.github/workflows/`. Commit after every task.
